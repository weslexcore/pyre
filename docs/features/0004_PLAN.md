# Feature Plan: Update UI Based on New Design System Guidelines with DaisyUI Migration

## Description
Update the UI components to align with the new design system guidelines outlined in the symbol usage, design system, and component usage rules. ## Technical considerations

### DaisyUI Configuration (following official guide):
- Use `@plugin "daisyui"` syntax in CSS imports
- Configure theme colors to match Pyre brand palette
- Set up component variants to use Pyre design system
- Ensure proper plugin order in Tailwind configuration
- Maintain existing custom utilities alongside DaisyUI components

### Symbol Sizing Algorithm:
- Default to large sizes (120px+) for maximum visibility
- Hero sections use extra-large (200px+) for dramatic impact
- Service cards use large (160px+) for clear identification
- Navigation elements use medium-large (120px+) for visibility
- Footer uses large (160px+) for accent elements

### DaisyUI Component Mapping:
- Replace custom card components with `card` and `card-body`
- Use `btn` variants for all buttons
- Replace custom form components with `form-control` and `input`
- Use `container` and `grid` for layout
- Implement `modal` or `drawer` for signup forms

### Color Combination Implementation:
- Use DaisyUI theme colors for consistent styling
- Configure DaisyUI theme to respect Pyre color rules
- Never mix more than 2 Pyre brand colors together
- Use DaisyUI color utility classes for combinations

### Typography Updates:
- Use golden ratio scale (1.618 multiplier) for type scaling
- Apply proper kerning based on font size
- Use DaisyUI typography utilities where applicable
- Maintain custom font families in DaisyUI theme

### Symbol Positioning Guidelines:
- Hero sections: Symbols as primary visual focus
- Service cards: Symbols prominently above titles
- Navigation: Symbols as primary visual indicators
- Footer: Symbols as accent elements with prominence

### Accessibility Requirements:
- Maintain WCAG AA contrast ratios
- Provide descriptive alt text for all symbols
- Ensure symbols enhance understanding, not replace text
- Test with screen readers for proper semantic meaning
- Use DaisyUI accessibility features

### Performance Considerations:
- Optimize symbol files for web delivery (under 20KB when possible)
- Use lazy loading for symbols below the fold
- Preload critical symbols used in hero sections
- Consider CSS sprites for multiple symbols if needed
- Ensure DaisyUI doesn't bloat bundle size

## Migration Patterns

### DaisyUI Component Usage:
```astro
<!-- Before: Custom card component -->
<Card class="group">
  <div class="p-6">
    <h3>Title</h3>
  </div>
</Card>

<!-- After: DaisyUI card component -->
<div class="card bg-base-100 shadow-xl">
  <div class="card-body">
    <h3 class="card-title">Title</h3>
  </div>
</div>
```

### Button Component Updates:
```astro
<!-- Before: Custom button variants -->
<Button variant="primary" size="lg">Click me</Button>

<!-- After: DaisyUI button variants -->
<button class="btn btn-primary btn-lg">Click me</button>
```

### Form Component Updates:
```astro
<!-- Before: Custom form components -->
<div class="form-group">
  <input type="email" class="form-input" />
</div>

<!-- After: DaisyUI form components -->
<div class="form-control">
  <input type="email" class="input input-bordered" />
</div>
```

### Symbol Usage Updates:
```astro
<!-- Before: Small accent symbols -->
<Symbol name="sauna" size="sm" className="opacity-80" />

<!-- After: Large prominent symbols -->
<Symbol name="sauna" size="xl" className="symbol-prominent" />
```

### Color Combination Updates:
```astro
<!-- Before: Mixed color usage -->
<section class="bg-background text-foreground">

<!-- After: DaisyUI theme colors -->
<section class="bg-base-100 text-base-content">
```
